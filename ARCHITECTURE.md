## Key Design Patterns

### 1. Service Layer Pattern
Each service encapsulates a specific domain:
- SMS operations
- Model operations
- RAG operations

### 2. Dependency Injection
Services are injected where needed:
```dart
ragService.initialize(modelManager: modelManager)
```

### 3. Async/Await Pattern
All I/O operations are asynchronous:
```dart
Future<List<SmsMessage>> getAllSms() async { ... }
```

### 4. Factory Pattern
Models are created from data:
```dart
SmsMessage.fromMap(map)
RagSearchResult.fromChunkSearchResult(result)
```

### 5. Builder Pattern
Complex queries are built step-by-step:
```dart
await ragService
    .search(query: q, limit: 5, maxDistance: 1.2)
```

## Benefits of This Architecture

1. **Modularity**: Each component has a single responsibility
2. **Testability**: Services can be mocked and tested independently
3. **Reusability**: Services can be used across different UIs
4. **Maintainability**: Changes are localized to specific modules
5. **Scalability**: Easy to add new data sources or models
6. **Type Safety**: Strong typing throughout the system
7. **Async Support**: Non-blocking operations for better UX
